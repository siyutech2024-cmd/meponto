# PontoSys 操作手册

PontoSys 是 meponto 用于站点运营、骑手管理、招聘转化、排班跟进、财务对账和总部巡查的运营系统。系统不替代 99 官方后台作为原始数据源；总部每天将前一天数据导入 PontoSys，站点在 PontoSys 内执行、记录和闭环。

## 1. 使用对象

- 总部运营：导入 D-1 数据、下发排班建议、风险预警、招聘缺口和巡查任务。
- 总部后台：处理绑定状态、数据核验、异常复核和跨站点数据支持。
- 站点负责人：承接总部任务，管理站点日流程、骑手执行、异常升级和收工复盘。
- 运营助理：维护签到、资料核验、装备检查、群通知、问题登记和系统更新。
- Leader：跟进首班、骑手日常表现、异常沟通和早期留存。

## 2. 每日数据导入

1. 总部从 99 后台数据源导出前一天数据。
2. 检查 Rider ID、手机号、Ponto、班次、在线时长、订单、AR、CAA、财务和异常字段。
3. 将数据导入 PontoSys。
4. 生成数据批次号，保留导入时间和负责人。
5. 系统生成异常队列，包括缺字段、重复、付款差异、未排班上线、低 AR、高取消和事故投诉。
6. 总部将异常分派给站点或总部后台处理。

## 3. 站点日任务

1. 站点负责人查看总部下发的当日任务。
2. 运营助理准备签到表、培训名单、装备检查、招聘到场名单和群通知。
3. 站点双人确认未结事故、投诉、付款争议、风险骑手和次日排班缺口。
4. 每个任务必须有负责人、时限、系统记录和闭环状态。

## 4. 全职骑手管理

- 班前 30 分钟完成到站签到和装备检查。
- 骑手日目标包括在线时长、OPH 1.5、AR 95%、高峰期在线和夜班定位。
- 午晚高峰禁止离线，必须在指定区域。
- 夜班每 10 分钟定位，危险区域拒单必须提交说明。
- 收工前核对在线时长、OPH、AR、事故、投诉、付款争议和次日安排。

## 5. 招聘转化

1. 总部市场统一发布招聘公告和物料。
2. 总部运营收集线索并安排线下注册。
3. 站点接待候选人，核验资料并协助 99 app 注册。
4. 总部后台处理绑定状态和数据核验。
5. 绑定成功后站点安排 Ponto、Leader、应用内聊天室和首班培训。
6. 首班后按 AR、OPH、在线时长、取消、投诉、事故判断通过、待观察或淘汰。

## 6. 站点运营

- 每站配置 2 人：站点负责人和运营助理。
- 总部提供 D-1 数据包、排班建议、风险预警和招聘缺口。
- 站点负责把总部数据转为动作：谁到站、谁补资料、谁首班、谁风险跟进、谁付款核查。
- 现场和数据不一致时，当天反馈总部复核。

## 7. 总部巡查

总部巡查不只看口头汇报，而是检查证据：

- 是否有系统记录。
- 是否有截图、签到表、群记录、定位、事件单或复盘表。
- 是否超时。
- 是否有负责人。
- 是否完成闭环。

核心指标：记录完整率 100%，证据可查率 95%+，超时动作 0，异常闭环率 100%。

## 8. 权限和红线

- PontoSys 后台仅限授权人员使用。
- 不在群内收集 CPF、地址、付款争议等敏感信息。
- 不向骑手展示总部后台、数据导入表或未确认的排班/付款数据。
- 不以 meponto 名义冒充 99 官方，不发布未经总部确认的政策。
- 事故、人身安全、严重投诉、付款争议、后台数据异常必须升级总部。
